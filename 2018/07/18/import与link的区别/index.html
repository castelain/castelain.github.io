@import与link的区别

  • 2018-07-18

前情提示

在 html 页面中加载 css 的四种方式：

  • 内嵌式

  • 导入式

  • 外链式

  • 内联式

引入的样式的优先级：内联 > 内嵌 > (外链或导入)
其中，外链和导入两种方式引入的样式表的优先级高低，取决于定义位置的先后顺序，最后定义的优先级最高。

@import 与 link 的区别

  1. 来源不同
    @import是由css提出的，只能用于导入 css 样式表；link是由 xhtml 提出的，不仅可以用于引入样式表，还能做其他的事：比如说定义RSS，定义rel属性等。
  2. 兼容性不同
    @import 是 css2.1 才提出来的，所以老的浏览器不支持它，要 IE5 以上才能支持它。而 link 是 xhtml 中的，所以并没有兼容性问题。
  3. 加载时间与顺序不同
    使用 @import 导入样式，浏览器会先加载页面结构，再导入样式。这种情况下，如果网速很慢，可能会出现页面结构出现在页面上了，可是样式没加载出来，页面显得比较难看的情形，会影响用户体验。而，使用 link 来加载样式表时，当用户要浏览某一页面时，浏览器会先将样式表加载进来、准备好，然后再加载页面具体内容与结构。这样的话，即使网络较慢，也不会影响页面的呈现效果了。
  4. dom控制不同
    js 不能操纵 @import 元素。而，js 可以操纵 link 这个 dom 元素。
