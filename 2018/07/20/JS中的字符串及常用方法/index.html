JS中的字符串及常用方法

  • 前端知识点
  • 2018-07-20

在 JS中用单双引号包裹起来的都叫字符串，它是由 0 到 多个字符组成的。

字符串具有一些有意思的特性：
1.它是以数字作为索引的，从零开始(以超出索引范围的数字下标去访问字符，得到的结果是 undefined )
2.字符串中有一个叫做 length 的属性，存储的是字符串的长度
toome8888

真实项目中，操作字符串的情况很常见。因此，需要掌握一些常用的字符串操作方法
console.dir(String.prototype)


charAt / charCodeAt

[str].charAt()：返回指定索引位置的字符
它与 str[索引] 的区别在于：当指定的索引不存在是，中括号方式返回的是 undefined，而 charAt() 方式返回的是空字符串

[str].charCodeAt()：在 [str].charAt() 基础上，将获取到的字符转换为 unicode 编码值（ 对应ASCII 码表 ）
部分ASCII 码表如下：

48-57 对应的是数字 0-9
65-90 对应大写的 A-Z
97-122 对应小写的 a-z
String.fromCharCode(十进制的 unicode 值) : 把值按照 ASCII 码表中的信息，转换为相应的字符

substr / substring / slice非常非常非常重要

这些方法都是用于实现字符串截取的
[str].substr(n,m)：从索引 n 开始，截取 m 个字符
[str].substring(n,m)：从索引 n 开始，截取到索引为 m 处的字符（不包含索引为 m 处的字符），把找到的字符截取
[str].slice()：和 substring 语法一样，区别在于 slice 方法支持用负数作为索引
当索引是负数的时候，浏览器在处理的时候，是用字符串的总长度加上索引，然后按照正数索引的情况进行处理

一些细节知识点：

  1. 对应[str].substr()/[str].substring()/[str].slice()，如果只传递了 n ，表示从索引 n 开始，一直截取到末尾
    2.如果传递的索引超出最大长度，也是返回能截取到的部分即可
    3.如果一个参数都不传递，相当于把整个字符串都截取，返回的是整个字符串（字符串的克隆）

toUpperCase / toLowerCase

[str].toUpperCase：把字符串中的所有字符都大写
[str].toLowerCase：把字符串中的所有字符都小写

indexOf / lastIndexOf

[str].indexOf()：检测当前字符在字符串中首次出现的位置的索引
[str].lastIndexOf()：检测当前字符在字符串中最后次出现的位置的索引

细节知识点：
如果 [str].indexOf(char) 等于 -1，就说明当前字符在该字符串中不存在（根据这个规律，可以验证当前字符串中是否包含某一字符）

split

[str].split()：按照某一字符，把字符串拆分成数组中的某一项，和数组中的 join() 方法是对应的（[arr].join()：是以指定字符将数组拼接成一个字符串）
拆分得到的数组的每一项都是字符串
支持正则
[str].split(‘ ‘)：以空格进行拆分
[str].split()：将整个字符串拆分为一个个的字符组成的数组

replace

[str].replace(old,new)：实现字符或字符串的替换（以新换旧）
执行一次 replace，只能替换一次。在不使用正则的情况下，有的时候需要执行很多次 replace方法。因此， replace 方法经常要和正则一起配合使用

trim / trimLeft / trimRight

[str].trim()：去除字符串开始和结尾的空格
[str].trimLeft()：去除字符串开始的空格
[str].trimRight()：去除字符串结尾的空格
