数组的基础结构

数组是对象数据类型的

数组也有属性名，不过属性名为数字。把数字属性名称为索引：数组把数字作为索引，索引从零开始，有一个表示数组长度的属性：length
类数组：类似于数组，但不是数组的数据
1.通过 getElementsByTagName() 获取的元素集合是类数组
2.函数中的实参集合 arguments 也是类数组
…

遍历数组

for 循环：只能遍历到数组私有的一些属性
for in 循环：可以把一些自定义的公共属性也遍历到
因此，真实情况下，常用的是 for 循环遍历数组

数组中的常用方法

数组中有很多的方法

数组中的方法的记忆点
1.方法的意义和作用
2.方法的形参
3.方法的返回值
4.通过此方法后，原来的数组是否发生了改变

数组的增删改

增加
push

作用：向数组的末尾追加内容
参数：一到多个，任何数据类型都可以。想要给数组末尾追加什么，直接传递到 push 方法中就可以了，传递的多个参数用逗号隔开
返回值：是新增数组的长度
原有数组改变了
等价于 array.splice(array.length,0,x)

unshift

作用：向数组开头追加新元素
参数：一到多个需要追加的元素，可以是任何数据类型
返回值：新增后数组的长度
原来数组改变了
等价于 array.splice(0,0,x)

array[length] = xxx

把数组当做一个普通的对象，使用对象的键值对的操作，给其设置新的属性（索引）
只能增加一个元素，且只能增加在数组末尾

删除
pop

作用：删除数组最后一项
参数：无
返回值：被删除的那一项
原来数组改变了
等价于 array.splice(array.length-1,1)

shift

作用：删除数组第一项
参数：无
返回值：被删除的那一项
原有数组改变了
注意：使用 shift 方法删除第一项后，后面每一项的索引都要向前进一位（导致后面的项的索引都发生了改变）
等价于 array.splice(0,1)

delete array[索引]

作用：删除指定索引位置对应的元素
把数组当做普通对象操作
当前项被删除后，原有数组其他项的索引不变，当前索引的 length 也不会改变
不建议使用

array.length --

作用：删除数组的最后一项元素

splice

总的作用：数组中内置的一个方法，可以实现数组元素的增删改

作用：1.实现删除：splice(n,m)：从索引 n 开始，删除 m 个
参数：m 不写表示删除到数组末尾；要是一个参数都不传递，就一个元素也不会删除
返回值：被删除的数组元素组成的一个新数组
原有数组改变了

作用：splice实现修改
splice(n.m,x) ：在原有删除的基础上，用 x 代替删除的内容

作用：splice实现增加
splice (n,0,x)：在修改的基础上，一项都不删除，把 x 插入到索引 n 的前面

array.splice(0,0,x)

在数组开始添加元素 x

array.splice(array.length,0,x)

在数组末尾添加元素 x

array.splice(0,1)

删除数组第一项元素

array.splice(array.length-1,1)

删除数组最后一项元素

数组的查询

slice

作用：数组的查询
参数：slice(n,m) -> 从索引 n 开始找到索引为 m 处（不包含 m 处的元素）
返回值：找到的部分组成的一个新数组
原来的数组不变

细节知识点：slice(n,m)
1.如果 m 值不写slice(n)，表示从索引 n 开始找到末尾
2.slice(0) / slice() ，表示数据克隆，克隆一份与原来数组一模一样的新数组
3.支持负数索引，如果传递的索引为负数，浏览器解析的时候是按照 length + 负数索引 来处理的
4.如果传递的索引不是有效数字，先 Number()，然后 parseInt()，结果为有效整数，则将其作为 n 来执行 array.slice(n)，否则执行 array.slice(0)
5.如果 n >= m，返回 空数组 []

数组的拼接

concat

作用：将多个数组拼接在一起
参数：0 - 多个要拼接的内容，可以是数组，也可以是一些数据值，多个值之间用‘，’隔开
返回值：拼接后得到的新数组
原有数组没改变

如果只写了array.concat() 则相当于只是进行了数组的克隆，返回了一个与原数组一模一样的新数组

数组转为字符串

toString

作用：实现把数组转为字符串（转换后的字符串以逗号分隔每一项）
参数：无
返回值：字符串
原有数组没改变

join

作用：把数组按照指定分分隔符转换为字符串，和字符串的 split 方法先相对应
参数：指定的连接符
返回值：转化得到的字符串
原有数组没变

数组的排列和排序

reverse

作用：将数组元素倒序排列
参数：无
返回值：原数组倒序排列得到的数组
原数组改变了

sort

作用：实现数组排序（递增或递减）
参数：无或一个回调函数
返回值：完成排序后的数组
原数组改变了

注意：不传递参数的情况下，默认是进行升序的排序。此时，数组元素为0-9以内的数字，则可正确进行排序。但是，如果有值大于9的数组元素，排序结果就会很诡异（只能按值得第一位进行排序）。此时，就必须通过传递回调函数来进行排序，可解决上面说的那个问题。

检查数组中是否包含某一项

indexOf / lastIndexOf

作用：检查指定元素是否在数组中，若存在则返回元素的索引，否则返回 -1
indexOf 返回指定元素首次出现的索引位置；lastIndexOf 返回指定元素最后出现的索引位置
参数：待查找的一个元素
返回值：元素索引或 -1
原数组没改变

注意：这两个方法在 IE6-8 是不兼容的；String 里的 indexOf 和 lastIndexOf 方法兼容所有浏览器。

数组的遍历

以下方法在 IE 6-8 下都不兼容
forEach
作用：遍历数组中的每一项
参数：一个回调函数 function（value, index）
返回值：无
原数组没改变

map

作用：遍历数组中的每一项，在 forEach 的基础上，可以修改每一项的值
参数：一个回调函数
返回值：修改后的数组
原数组没改变

filter

作用：创建一个新数组，新数组中的元素是通过检查了的指定数组中符合条件的所有元素
参数：一个回调函数
返回值：一个数组（满足条件的数组元素组成的新数组）
原有数组没改变

find

作用：返回通过测试（函数内判断）的数组的第一个元素的值
参数：一个回调函数
返回值：一个数组元素
原数组没有改变

reduce

作用：接受一个函数作为累加器，计算数组中的所有元素累加的结果
参数：一个累加器（回调函数）
返回值：累加结果（一个值）
原数组没改变

every

作用：检测数组中的所有元素是否都符合指定条件（通过函数提供）
参数：一个回调函数
返回值：true / false
原数组没有改变

some

作用：检测是否数组中有满足指定条件（函数决定）的数据项
参数：一个回调函数
返回值：true / false
原数组没改变

数组去重

基本方法（双重循环、比较去重）

遍历数组中的每一项，拿每一项和后面的项依次进行比较。如果相同，则把相同的这项从原有数组中删除

性能不佳，代码编写要注意，否则可能会出现数组塌陷问题

数组塌陷问题：使用 splice 方法删除数组中的某一项后，删除了的这一项的后面的所有项的索引，都要向前进一位（在原有索引上减一）。此时如果 j ++ ，循环操作的值累加了，再通过最新 j 获取的元素就不是紧挨这一项的元素了，而是跳过一项获取的元素。
–》 解决方法：splice 后，先 j -- ，再 j ++，此时 j 还是原有索引。再获取的时候，获取到的就是紧挨着删除了的那项的元素
–》或者，改写内层 for 循环，删除迭代条件，在循环体里进行判断：删除元素后，不j ++；没删除元素，就 j ++

indexOf 处理

通过 indexOf 方法，查看当前元素是否存在于后面的数组项组成的新数组中，若存在，就删除当前元素
优点：性能高于 双重循环 的方式
缺点：IE 6-8 不兼容 indexOf 方法

对象键值对处理

1.创建一个空对象，将数组中的每一个元素都作为属性名和属性值，插入到对象中
2.在每一次插入到对象中之前，都要进行判断：如果该元素形成的属性在对象中已存在，则不插入；否则，则将其插入到对象中
优点：性能较好，无兼容性问题

相邻比较法

1.先将数组进行排序
2.相邻比较：如果相等，则删掉其中某一项
3.重复步骤2，直至比较到最后一项

数组排序

数组中的 sort 方法

实现数组排序，最常使用的就是数组中的 sort 方法。但是，其他算法实现的数组排序也需要了解掌握，作为一种开拓思维的培养

冒泡排序

所谓冒泡，指的是每经过一轮排序后，可将当前数组元素中的最小值或最大值给放在数组的后面
特点：
1.冒泡排序中，共需要进行 [array].length - 1 轮的循环（理解：每轮找出一个最值，最后一个值就是最小值了，共需 length - 1 轮 就可完成数组排序了）
2.每轮需要进行（ [array].length - 1 - 已完成的轮数 ）的比较（理解：对于一个数组元素而言，它不需要和自己进行比较，也不需要之前轮数中找出的最值进行比较，因此，所需的比较轮数就是 [array].length - 1 - 已完成的轮数 ）

快速排序

基本思路就是，
1.找出数组中的中间那一项；
2.将数组中剩下的元素和找出的那一项中间结果进行比较：若大于中间元素，则放在左边的一个新数组中；否则，就放在右边的一个新数组中。（这样一轮下来，左边数组的元素都比中间元素小，右边数组的元素都比中间元素大或相等）
3.将步骤2得到的新数组作为数据，重复步骤1和步骤2，直至得到每个数组的长度都等于1或0
4.最后将排序过程中得到的所有左右数组和中间元素都连接起来，即可得到排序的结果（左边数组 + 中间元素 + 右边数组）

插入排序

和生活中，打扑克时候的摸牌和理牌的思路与情境类比，就比较好理解插入排序的主要思路了
假设现在手中已经摸了n 张牌（手中初始为1张牌，即第一个数组元素组成的一个新数组），而这 n 张牌，按照后面的步骤整理，已经是从小到大的了。
此时，又摸了一张牌 A。这时，用这张牌与手中的 n 张牌进行比较，从右往左看起，知道看到一张牌比 A 小，假设找到的牌是第 k 张。则，要把 A 插入到第 k 张牌的后面
一种比较特殊的情况：用 A 去和手中的牌比较的时候，发现每一张牌都比 A 要大（即比较到了第一张牌，也没找到一张比 A 小的牌），此时，需要把 A 放在第一张牌的前面
不停的摸牌，当摸到最后一张的时候，手中的牌也都是从小到大升序排列的了（类似于最后得到了一个升序排列的数组）
