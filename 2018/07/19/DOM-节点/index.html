DOM-获取元素的方法

  • 前端知识点
  • 2018-07-19

DOM 的节点

node : 节点，在一个 html 页面中的所有内容都是节点（包括标签、注释、文字文本等）

  • 元素节点： html 标签
  • 文本节点：文字内容（高版本浏览器会把空格和换行也当做文档节点）
  • 注释节点：注释内容
  • document 文档节点
  • …

元素节点

nodeType：1
nodeName：大写的标签名（在部分浏览器的怪异模式下，我们写的标签名是小写，它获取的就是小写，反之为大写）
nodeValue：null

[currentElement].tagName：获取当前元素的标签名（获取的标签名一般都是大写）

文本节点

nodeType：3
nodeName：#text
nodeValue：文本内容

注释节点

nodeType：8
nodeName：#comment
nodeValue：注释内容

文档节点

nodeType：9
nodeName：#document
nodeValue：null

节点是用来描述页面中每一部分之间关系的

DOM 节点关系属性

childNodes

获取当前元素的所有的子节点（子节点集合：类数组）
注意：不仅仅是元素子节点，文本、注释等都会包含在内;
子节点说明只是在儿子辈分中查找;

children

获取所有的元素子节点（元素集合）
在 IE6-8 下获取的结果和标准浏览器中有区别（IE6-8 中会把注释节点当做元素节点获取到）

parentNode

获取当前元素的父节点（元素对象）

previousSibling / nextSibling

previousSibling：获取当前节点的上一个哥哥节点，不一定是元素节点，也可能是文本或注释
nextSibling：获取当前节点的下一个弟弟节点，获取的节点情况同上

previousElementSibling / nextElementSibling

previousElementSibling：获取当前节点的上一个哥哥元素节点
nextElementSibling：获取当前节点的下一个弟弟元素节点
IE6-8 下不兼容

firstChild / lastChild

firstChild：当前元素所有子节点中的第一个（不一定是元素节点，也可能是文本或注释）
lastChild：当前元素所有子节点中的最后一个（不一定是元素节点，也可能是文本或注释）

firstElementChild / lastElementChild

IE 6-8 下不兼容
firstElementChild：当前元素下的所有元素子节点的第一个节点
lastElementChild：当前元素下的所有元素子节点的最后一个节点

DOM 的增删改操作

真实项目中，偶尔会在 js 中动态创建一些 html 标签，然后把其增加到页面中

document.createElement

在 js 中动态创建一个 html 标签

appendChild

容器.appendChild(新元素)
把当前创建的新元素添加到容器的末尾

insertBefore

容器.insertBefore(新元素,老元素)
在当前容器中，把创建的新元素添加到老元素之前

removeChild

容器.removeChild(元素)
在当前容器当中，把某一个元素移除掉

replaceChild

容器.replacechild(新元素，老元素)
在当前容器中，用新元素替换老元素

cloneNode

元素.cloneNode(false / true)
把原有元素克隆一份一模一样的
false：只克隆当前元素本身
true：把当前元素以及元素的所有后代都进行克隆（深度克隆）
